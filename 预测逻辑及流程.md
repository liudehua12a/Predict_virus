二、预测目标
预测 三种玉米叶斑病 的发病程度：

病害	disease_key	预测目标
灰斑病	gray	发病株率(incidence) + 病情指数(index)
大斑病	blight	发病株率(incidence) + 病情指数(index)
白斑病	white	发病株率(incidence) + 病情指数(index)
风险等级划分 (a_config.py:72-82):

低风险: 值 ≤ 30
中风险: 30 < 值 ≤ 70
高风险: 值 > 70
三、预测流程详解
1. 数据准备阶段
① 历史天气数据补齐 (l_history_padding_and_prediction_runner.py)


从 weather_daily 表读取最近 21 天历史天气
如果不足 21 天 → 用 mock 数据向前补齐（以最早真实数据为模板复制）
② 未来天气预报获取


从 weather_daily 表读取未来 7 天预报数据
（数据来源：和风天气 API，已提前入库）
③ 特征工程 (i_online_prediction_preparation.py)


# 构造两类特征：

# A. 时序特征 (weather_seq_21)
# 最近 21 天的逐日气象数据组成的序列 (21天 × 24个气象特征)
# 包括：风速、降水、湿度、气温、土壤湿度、地表温度、气压、辐射等

# B. 过程特征 (BASE_MODEL_FEATURES)
# 派生统计指标，例如：
# - 累计降水 (3d/7d/14d/21d)
# - 连续高湿天数
# - 连续高温天数
# - 强降雨次数
# - 高温高湿天数
# - 有效积温 GDD
2. 滚动预测核心逻辑
核心思想：逐日滚动预测，用前一天的预测值作为下一天的输入


Day 1: 使用真实调查值(或零初始化)作为 previous_targets → 预测 Day1
Day 2: 使用 Day1 的预测值作为 previous_targets → 预测 Day2
Day 3: 使用 Day2 的预测值作为 previous_targets → 预测 Day3
...
Day 7: 使用 Day6 的预测值作为 previous_targets → 预测 Day7
预测起点确定 (n_online_prediction_service.py:38-93):


优先级：
1. 昨天真实调查值 → 使用 observation 作为起点
2. 昨天预测值 → 使用 prediction 作为起点  
3. 都没有 → zero_init (全零初始化)

预测计算 (d_model_training_testing.py:236-255):


# 模型输出的是增量 delta
delta = model(seq_input, tab_input)

# 应用 alpha 系数平滑
values = previous_targets + alpha * delta

# 单调性约束：病害程度不会倒退
if is_monotonic:
    values = max(values, previous_targets)

# 裁剪到 [0, 1] 范围
values = clip(values, 0, 1)
4. 预测结果存储
数据库表结构 (k_weather_data_storage.py:582-613):


CREATE TABLE disease_prediction (
    prediction_id INTEGER PRIMARY KEY,
    prediction_run_id TEXT,        -- 预测批次ID
    site_id INTEGER,               -- 点位ID
    batch_id INTEGER,              -- 批次ID
    model_type TEXT,               -- 模型类型
    
    predict_date TEXT,             -- 预测日期
    
    -- 灰斑病
    gray_incidence REAL,           -- 发病株率
    gray_index REAL,               -- 病情指数
    gray_risk_level TEXT,          -- 风险等级
    
    -- 大斑病
    blight_incidence REAL,
    blight_index REAL,
    blight_risk_level TEXT,
    
    -- 白斑病
    white_incidence REAL,
    white_index REAL,
    white_risk_level TEXT,
    
    is_current INTEGER,            -- 是否当前有效版本
    ...
);
版本管理：

每次新预测会生成新的 prediction_run_id
旧版本记录 is_current = 0，新版本 is_current = 1
